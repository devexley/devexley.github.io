Ruby Arrays & Hashes

The Difference

June 25, 2015

Arrays & Hashes are collection objects used to store and retreive data. In general, an array is like a numbered list of items. The number is the key and the item is the value for that key. This makes arrays sequential in the way they store data. Hashes allow a key-value pairing, meaning you can create any kind of unique key (it does not have to be an integer) and have an associated value. This makes hashes non-sequential in the way they store data.

Here is a simple example of an array declaration.

array = ["ham", "bread", "mustard"]

The integer key starts as 0,1,2... and so on for each element. Since the key for each element (item) is an integer you can access the value by the following:

array[0]

So array[0] would give the value "ham" while array[2] would give the value "mustard".

Hashes are more like a dictionary in the way they work. The key is a look up for the value like a word is for a definition. Both can be any value type. Here is a simple declaration of a Hash.

sounds = {dog => "bark", cat => "meow", cow => "moo"}

Notice that we are declaring a bunch of key-value pairs. In this example a key would be "dog" and its value would be "bark". To access the value you would use the following:

sounds["dog"]

So when should you use each? An array should be used for lists which only need an integer index to iterate through. It is useful for storing values sequentially without any association. A hash on the other hand can be used to store associations such as the example given above. If I wanted a collection that stores relationships I would use a hash. But if I wanted to just have a list of things than I would use an array. So this should give you a general primer of the difference between arrays and hashes.
